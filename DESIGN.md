# BRUTALIST DESIGN GUIDE

この文書は、Brutalist デザインを人間と AI の両方に同じ解像度で伝えるためのガイドです。  
「無骨な見た目を真似ること」が目的ではありません。**情報構造がすぐ読めて、判断が速いUI** を再現可能にすることが目的です。

---

## 1. Brutalist とは何か

### 1-1. 一言で

装飾で雰囲気を作るのではなく、**構造・境界・文字・コントラストで意味をはっきり見せる**デザインです。

### 1-2. 何を目指しているか

- ユーザーが重要情報を一瞬で見分けられる
- 操作できるものと、読むだけのものがすぐ区別できる
- 状態変化が明確で、迷いなく次の行動に移れる
- 画面をまたいでも同じ設計思想が保たれる

### 1-3. 何を目指していないか

- 太い線を増やすだけ
- 全部を黒く重くするだけ
- レトロっぽく見せるだけ
- 可読性やアクセシビリティを犠牲にする

### 1-4. よくある誤解

| 誤解 | 実際 |
|------|------|
| 攻撃的で派手な見た目 | 派手さよりも情報構造の明快さが中心 |
| 色数を減らせば成立する | 色数より意味の割り当てが重要 |
| 何でも四角くして太線にする | 線の太さや形は階層を示すために使い分ける |

## 2. Visual Theme

### 2-1. 画面の空気感

- クリーンだが緊張感がある
- 編集ツールや作業環境のような道具感
- 装飾的ではなく、構造的
- 押す場所と読む場所が明快

### 2-2. 画面全体の構成原則

- 背景はフラットではなく、グリッドや区画線で秩序を表現する
- セクションは余白だけでなく、**境界と面**で分離する
- 強調は少数に限定し、画面全体を叫ばせない

---

## 3. Color

色は雰囲気のためではなく、**意味のために**使います。

### 3-1. Color Roles

| Role | 意味 | Light | Dark |
|------|------|-------|------|
| Canvas | 画面全体の基調面 | `#f5f3ee` | `#171717` |
| Panel | パネル・カードの面 | `#ffffff` | `#0a0a0a` |
| Ink | 主文字・主境界 | `#000000` | `#ffffff` |
| Emphasis | 強調・CTA・ハイライト | `#ffe500` | `#ffe500` |
| Destructive | 削除・危険操作 | `#ff0000` | `#ff0000` |
| Muted | 補助情報・弱い面 | `#e8e6e1` | `#2a2a2a` |

### 3-2. 使い方のルール

- 同じ意味に複数の色を割り当てない
- Emphasis（黄）と Destructive（赤）を「どちらも重要そうな色」として併用しない
- 文字色・背景色・境界色のコントラストを常に保つ
- Muted は主役にしない。補助情報と弱い面のみに使う

---

## 4. Typography

文字は装飾ではなく、**役割を伝えるUI部品**として扱います。

### 4-1. Font Family

| 役割 | フォント |
|------|---------|
| Heading | Bricolage Grotesque, Inter, sans-serif |
| Body / Label | Inter, system-ui, sans-serif |
| Mono / Tag | Space Mono, monospace |
| Serif（特別演出のみ） | Playfair Display, Georgia, serif |

### 4-2. Font Size Scale

| Step | Size | 用途 |
|------|------|------|
| xs | 11px | タグ・補助ラベル |
| sm | 13px | キャプション・メタ情報 |
| base | 15px | 本文・通常ラベル |
| md | 18px | 小見出し・強調本文 |
| lg | 22px | セクション見出し |
| xl | 28px | ページ見出し |
| 2xl | 36px | 大見出し |
| 3xl | 48px | ヒーロー・ランディング |

### 4-3. Font Weight

| 用途 | Weight |
|------|--------|
| 本文 | 400 |
| 機能ラベル | 500 |
| 小見出し | 600 |
| 大見出し | 700–800 |

### 4-4. Letter Spacing

| 用途 | 値 |
|------|-----|
| 本文 | 0em |
| 機能ラベル (label) | 0.06em |
| 見出し (heading) | 0.08em |
| 導入ラベル (kicker) | 0.12em |

機能ラベルは uppercase + 高めの letter-spacing でUI部品として見せる。  
本文は読みやすさ優先で、letter-spacing は原則 0 に保つ。

### 4-5. 役割ごとの構成例

```
SECTION LABEL      ← kicker: 13px / uppercase / 0.12em / mono
見出しタイトル      ← heading: 28px / bold / Bricolage Grotesque
本文テキストが入ります。 ← body: 15px / regular / Inter
```

## 5. Layout

レイアウトでは、余白より先に**構造**を決めます。

### 5-1. 設計の順序

1. 主従関係を決める
2. 区画を境界と面で分ける
3. 余白で関係性を示す
4. 装飾は最後に、必要な分だけ

### 5-2. Spacing Scale

| Step | Size | 主な用途 |
|------|------|---------|
| 1 | 4px | アイコンとラベルの間など極小余白 |
| 2 | 8px | インライン要素間 |
| 3 | 12px | コンポーネント内パディング |
| 4 | 16px | カード内部・フォームグループ間 |
| 5 | 24px | セクション内の要素間 |
| 6 | 32px | セクション間（グリッドベース） |
| 7 | 48px | 主要セクション間 |
| 8 | 64px | ページレベルの区切り |

余白は**雰囲気づくりではなく関係性の表示**として使う。近い要素は関係あり、遠い要素は関係なし、を視覚的に示す。

### 5-3. Grid

- グリッドは装飾ではなく、秩序の可視化
- 基本単位: 32px
- 常時強く見せず、背景として薄く支える

---

## 6. Surface, Border, Radius

Brutalist では「面」と「境界」が特に重要です。

### 6-1. 面の考え方

- **Canvas**: 全体の土台
- **Panel**: 独立した情報のかたまり
- 面が異なるなら、境界か背景差のどちらかを必ず入れる

### 6-2. Border Width

| 用途 | 値 |
|------|-----|
| 小さなコントロール（input, checkbox 等） | 2px |
| カード・ダイアログ・ボタン | 3px |
| 区画分離・トップバー・レール | 4px |

Border color は原則 Ink（`#000000` / `#ffffff`）。Muted border は Ink の 30% 程度。

### 6-3. Radius

- 原則: `border-radius: 0`（rounded-none）
- 丸みは原則として付けない
- 形そのものに意味がある場合（chip、badge、avatar など）のみ例外を認める

---

## 7. State & Interaction

状態変化は、色だけでなく**境界・濃度・文字・反転**でも伝えます。

### 7-1. 必須の状態

| 状態 | 説明 |
|------|------|
| default | 通常表示 |
| hover | マウスオーバー |
| active / pressed | 押下中 |
| focus | キーボードフォーカス |
| disabled | 操作不可 |
| error / invalid | 入力エラー |
| pending / loading | 処理中 |

### 7-2. 各状態の視覚変化仕様

**Button（primary）**

| 状態 | 背景 | 文字 | 境界 | 補助 |
|------|------|------|------|------|
| default | Ink | Ink-inverse | Ink 3px | none |
| hover | Emphasis | Ink | Ink 3px | 必要なら軽微な位置変化 |
| active | Ink | Ink-inverse | Ink 3px | 必要なら押下位置変化 |
| disabled | Muted | Muted-text | Muted 2px | none |
| focus | Ink | Ink-inverse | Emphasis 3px | outline で可視化 |

**Input**

| 状態 | 境界 | 背景 | その他 |
|------|------|------|--------|
| default | Ink 2px | Panel | — |
| hover | Ink 3px | Panel | — |
| focus | Ink 3px | Panel | アウトライン: Emphasis 2px offset |
| error | Destructive 3px | Panel | エラーメッセージ表示 |
| disabled | Muted 2px | Muted | opacity: 0.5 |

**Card / Panel**

| 状態 | 境界 | その他 |
|------|------|--------|
| default | Ink 3px | — |
| hover（clickable） | Ink 3px | 必要なら translate(-1px, -1px) |
| active（clickable） | Ink 3px | 必要なら translate(1px, 1px) |

### 7-3. ルール

- hover で少し変わる（境界・背景・位置のいずれかで反応を見せる）
- pressed で明確に反応する（位置や濃度の変化で反応を見せる）
- disabled は opacity と Muted color の組み合わせで、hover と明確に区別する
- focus は必ず**色以外**の手段（outline、境界太さ、位置）でも視認できるようにする
- Destructive アクションは通常ボタンと同じ視覚的テンションにしない

### 7-4. 禁止

- 色だけで状態差を出す
- disabled なのに hover と見分けがつかない
- Destructive が Emphasis（黄）と同じ文脈で見える
- focus を非表示にする（`outline: none` を理由なく使う）

## 8. Mobile

PC と同じ設計思想を保ちつつ、タッチ操作と小画面に対応します。

### 8-1. 基本方針

- **構造は保つ、レイアウトは変える**  
  モバイルでも境界・面・コントラストによる意味付けはそのまま維持する。  
  変えるのはカラム数、サイドレールの扱い、フォントサイズのみ。

- **タッチターゲットは最低 44px × 44px**  
  視覚的に小さく見せても、タップ領域は十分に確保する。

- **トップバーは必ず固定（position: sticky）**  
  スクロール中でもナビゲーションを失わせない。

### 8-2. ブレークポイント

| Name | px | 対象 |
|------|-----|------|
| sm | 640px | 縦スマートフォン |
| md | 768px | 横スマートフォン・小タブレット |
| lg | 1024px | タブレット・小型ノートPC |
| xl | 1280px | デスクトップ |
| 2xl | 1536px | 大型ディスプレイ |

### 8-3. 構造の変換ルール

| 要素 | Desktop | Mobile |
|------|---------|--------|
| サイドレール | 固定表示 | 非表示 → ハンバーガーまたはボトムシート |
| カラム数 | 2–4 | 1（縦積み） |
| トップバーの高さ | 56px | 52px |
| 見出しサイズ | xl〜2xl | lg〜xl（1〜2段階縮小） |
| カード padding | 24px | 16px |
| Border width | そのまま | そのまま（変えない） |

### 8-4. モバイルでの禁止

- hover 状態だけに情報を隠す（タッチデバイスには hover がない）
- タップ領域が 44px 未満のボタン・リンク
- 全幅ではないモーダルを画面中央に配置する（モバイルはボトムシートか全幅）
- 固定フォントサイズをモバイルに持ち込む（最低 14px / 推奨 15px 以上）

---

## 9. Do / Don't

### Do

- 構造が先、装飾は後
- 強調は少数に絞る
- 色・境界・文字の意味を固定する
- 見た瞬間に優先順位が読める状態を目指す
- Destructive 操作は視覚的にも操作的にも通常フローから分離する
- focus と disabled を必ず視覚的に区別する

### Don't

- 全部を強調して階層を壊す
- 画面ごとに色の意味を変える
- Brutalist を「荒っぽい見た目の模倣」にする
- 可読性を下げてでも雰囲気を優先する
- hover 情報だけに頼る（モバイルで消える）
- `outline: none` を理由なく書く

### レビュー時の確認項目

1. 重要情報の優先順位が一瞬で分かるか
2. 操作可能要素が、読むだけの要素と混ざっていないか
3. 状態差分が色以外でも判別できるか（境界・outline・位置で区別できるか）
4. Emphasis と Destructive の意味が混線していないか
5. 装飾を全部取り除いても構造が残るか
6. モバイルで hover に依存している情報がないか
7. タッチターゲットが 44px 以上確保されているか
8. focus が常に視認できるか
